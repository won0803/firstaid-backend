# 🚒 소방 공공 데이터 활용 백엔드 API 서버

공공 데이터 포털의 소방 및 응급처치 관련 데이터를 활용하여, 필요한 데이터를 정제하고 효율적으로 조회할 수 있도록 구현한 백엔드 API 서버입니다. 대용량 공공 데이터를 다루는 가공 프로세스와 API 최적화에 초점을 맞춰 개발했습니다.

---

## 🛠 사용 기술
* Java 17 / Spring Boot / Spring Data JPA
* MySQL
* Gradle
* Python (데이터 초기 정제 및 마이그레이션)
  
---

## 🔥 핵심 구현 및 고민한 점

### 1. 외부 데이터 정제 및 파이썬을 활용한 DB 초기 구축
* 공공 데이터 포털에서 제공하는 원본 데이터(엑셀/XML 등)의 형식을 통일하기 위해, **Python 스크립트를 직접 짜서 데이터를 정제하고 MySQL 데이터베이스에 밀어 넣는 초기 마이그레이션 작업을 수행**했습니다.
* 이후 스프링 백엔드 환경에서 원활하게 엔티티(Entity)로 변환해 활용할 수 있도록 가공 프로세스를 고민했습니다.

### 2. 조건별 데이터 필터링 및 대량 조회 최적화
* 소방서 위치나 응급처치 정보 등 사용자가 원하는 조건에 맞는 데이터를 정확하게 필터링하여 응답하는 API를 구현했습니다.
* 많은 양의 공공 데이터가 한 번에 조회될 때 서버와 DB에 부담이 가지 않도록 페이징(Pagination) 처리를 적용하여 안정적인 데이터 전송 환경을 만들었습니다.

### 3. 예외 처리(Exception Handling)와 데이터 검증
* 외부 데이터 특성상 비어있거나 형식이 유효하지 않은 값이 들어오는 경우를 대비하여, 서비스 계층에서 엄격하게 데이터를 검증(Validation)하는 로직을 추가했습니다.
* 예상치 못한 데이터 오류가 발생하더라도 서버가 다운되지 않고 클라이언트에게 정확한 에러 메시지를 반환하도록 예외 처리 시스템을 구축했습니다.
