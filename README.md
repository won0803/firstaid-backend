# 🚒 소방 공공 데이터 활용 및 시각화 백엔드 API 서버

공공 데이터 포털의 소방 및 응급처치 관련 데이터를 수집·가공하고, 이를 위치 기반 API와 연동하여 효율적으로 조회할 수 있도록 구현한 백엔드 API 서버입니다. 대용량 비정형 데이터를 정제하는 자동화 ETL 파이프라인 구축과 카카오 API 기반의 위치 데이터 최적화에 초점을 맞춰 개발했습니다.

---

## 🛠 사용 기술
* Java 17 / Spring Boot / Spring Data JPA
* MySQL
* Gradle
* Python (데이터 초기 ETL 및 마이그레이션)
* Kakao Local API (주소 변환 및 위치 기반 데이터 처리)
  
---

## 🔥 핵심 구현 및 트러블슈팅

### 1. 비정형 데이터 정제 및 자동화 ETL 파이프라인 구축
* 공공 데이터 포털의 수만 건에 달하는 원본 비정형 소방 출동 데이터를 정제하기 위해, Python 스크립트를 활용한 자동화 가공 프로세스를 구축했습니다.
* 원본 엑셀 및 XML 데이터를 파싱하여 일관된 스키마로 표준화한 후 MySQL 데이터베이스에 안정적으로 적재(Migration)하였으며, 스프링 백엔드에서 객체지향적인 엔티티(Entity)로 유연하게 활용할 수 있도록 기반을 다졌습니다.

### 2. 카카오맵(Kakao Maps) 연동 및 위치 데이터 좌표 오차 보정
* 텍스트 기반의 소방 위치 정보를 프론트엔드의 Kakao Maps API 상에 정확한 마커로 시각화할 수 있도록, 위·경도 좌표 데이터 변환 및 정제 작업을 수행했습니다.
* 연도별·지역별 국토 좌표계 기준(EPSG:5174) 차이로 인해 일부 마커가 바다 한가운데나 엉뚱한 위치에 찍히는 오차 누적 현상을 발견하고 이를 정교하게 분석했습니다.
* 변환 타겟 좌표계를 세분화하여 동적으로 보정하는 알고리즘을 반영함으로써, 카카오맵 상에 데이터가 왜곡 없이 올바르게 시각화되도록 공간 데이터의 무결성을 확보했습니다.

### 3. 복합 조건 필터링 및 대량 데이터 조회 최적화
* 사용자가 원하는 소방서 위치, 응급처치 지표 등 다양한 요구 조건에 맞춰 정확한 데이터를 필터링하고 빠르게 응답하는 RESTful API를 구현했습니다.
* 수만 건의 데이터가 집중되는 대량 조회 요청 시 발생할 수 있는 서버 및 DB 부하를 차단하기 위해, 스프링 데이터 JPA 기반의 페이징(Pagination) 처리를 적용하여 메모리 누수를 방지하고 데이터 전송 환경을 최적화했습니다.

### 4. 엄격한 데이터 검증(Validation) 및 글로벌 예외 처리
* 외부 공공 데이터 특성상 발생하는 Null 값이나 유효하지 않은 형식을 방지하기 위해 서비스 계층 전반에 엄격한 데이터 검증 로직을 도입했습니다.
* 예기치 못한 비즈니스 예외 상황에서도 서버의 가용성을 유지하고, 클라이언트에게 일관되고 명확한 메시지를 반환하도록 스프링의 `@RestControllerAdvice` 기반 예외 처리 시스템을 구축했습니다.
